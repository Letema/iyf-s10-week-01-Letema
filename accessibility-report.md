# Accessibility Audit Report

## 1. Issues Found

### Images
- Some images did not have proper descriptive alt text.

### Headings
- Heading structure was checked to ensure proper hierarchy (h1 → h2 → h3).

### Links
- Some links were updated to have more descriptive text instead of generic wording.

### Language Attribute
- The lang="en" attribute was confirmed in the HTML document.

### Form Labels
- All form inputs were checked to ensure they have proper associated <label> elements.

---

## 2. Fixes Applied

- Added meaningful alt text to all images
- Ensured proper heading structure (h1 for title, h2 for sections)
- Improved link text for clarity and accessibility
- Verified lang="en" is present in <html> tag
- Confirmed all form inputs have linked labels using for/id

---

## 3. Tools Used

- Chrome DevTools Lighthouse (Accessibility audit)
- Manual inspection of HTML structure
- WAVE Accessibility Checker (optional)

---

## 4. Final Result

- Improved accessibility score after fixes
- Better screen reader compatibility
- Improved semantic structure and usability
